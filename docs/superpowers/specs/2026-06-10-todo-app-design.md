## 아키텍처

단일 `index.html` 파일로 구성. `<head>`에 Tailwind CDN, `<body>`에 UI 마크업, `<script>`에 상태 및 로직을 모두 포함한다.

### 상태 구조

```js
let todos = []
// todo 항목: { id: number, text: string, done: boolean }

let filter = 'all' // 'all' | 'active' | 'completed'
```

상태가 변경될 때마다 `render()` 함수 하나가 전체 목록을 다시 그린다. 단방향 데이터 흐름으로 예측 가능한 상태 관리를 유지한다.

## 기능 명세

### 추가
- 입력창에 텍스트 입력 후 `Enter` 또는 `추가` 버튼 클릭으로 항목 추가
- `id`는 `Date.now()`로 생성, `done`은 `false`로 초기화

### 완료 토글
- 체크박스 클릭 시 `done` 값을 반전
- 완료된 항목은 텍스트에 취소선 스타일 적용

### 수정
- `수정` 버튼 클릭 시 해당 항목 텍스트가 인라인 `<input>`으로 전환
- `Enter` 키 또는 포커스 아웃(`blur`)으로 저장
- `Esc` 키로 수정 취소, 원래 텍스트 유지

### 삭제
- `삭제` 버튼 클릭 시 해당 항목을 `todos` 배열에서 제거 후 즉시 렌더

### 필터
- `전체`: 모든 항목 표시
- `미완료`: `done === false` 항목만 표시
- `완료`: `done === true` 항목만 표시
- 활성 필터 탭은 시각적으로 강조

## 엣지 케이스 처리

| 상황 | 처리 방식 |
|------|-----------|
| 빈 문자열 입력 | 추가 무시 |
| 공백만 입력 | `trim()` 후 빈 문자열이면 무시 |
| 수정 중 빈 값으로 저장 | 수정 취소, 원래 텍스트 유지 |
| 수정 중 `Esc` | 수정 취소, 원래 텍스트 유지 |
| 항목 없을 때 | "할 일이 없습니다" 빈 상태 메시지 표시 |
| 필터 결과 없을 때 | "해당하는 항목이 없습니다" 메시지 표시 |
